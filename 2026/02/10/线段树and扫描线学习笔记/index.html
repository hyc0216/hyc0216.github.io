线段树and扫描线学习笔记

线段树

一种能在O(log)复杂度下维护区间修改、区间查询的算法
需要实现5个主要的函数

pushup

根据儿子的数值修改父亲的数值
当儿子的数值被更新时
需要pushup更新父亲的值

pushdown

当需要向下递归修改或查找儿子的数值时
需要pushdown下放懒标记

注意：
在扫描线算法中不需要此函数
我们只保证心根节点的正确性

e.g.
有一个[1,6]的区间

  1. 对区间[1, 3]+1
    此时[1, 3].cnt == 1

  2. 对区间[2,6]+1
    此时会递归到区间[1,3]
    如果此时使用 pushdown 下放懒标记
    则 [1, 2].cnt == [1.3].cnt == 1
    [3, 3].cnt == [1,3].cnt + 1 == 2

  3. 对区间[1, 3]-1
    此时只会更改区间[1, 3]的cnt
    则[1,3].cnt = 0
    但是并不会向下递归修改区间[1,2][3,3]的cnt
    此时区间[1,2].cnt = 1, [3,3].cnt = 2
    这不对罢
    如果硬要递归，时间复杂度退化为暴力（TLE欢迎你）

  4. 对区间[2, 6]-1
    其实早就炸了
    此时[1, 3].cnt = 0
    所以pushdown没用
    别的先不看
    就看最后
    操作都全结束了[1,2].cnt和[3,3].cnt还tm＞0

build

建树的函数
一定要记得在main函数里调用啊
一定要给非叶子结点赋上左右端点啊
一定记得在递归之后pushup给当前节点赋初值啊
简单
但容易出错

modify

区间修改
正常线段树要在递归前pushdown下传懒标记
在递归后pushup更新该节点的值

query

区间查询
正常线段树要在递归前pushdown下传懒标记
但是因为并没有更新子节点的值
所以在递归之后不需要pushup更新当前节点的值

然后没了

扫描线

求若干个矩形的面积并
难点刚刚已经说了
整体思路就是先离散化，然后去重、排序
最后把一个矩形的左边界抽象成在区间 [y1, y2]上 +1
右边界抽象成在区间 [y1, y2]上 -1
从左到右扫描一遍
用线段树维护每一小段的长度
把面积依次加起来
就得到了答案

离散化、去重很ex
建议扔给vector
STL大法好

OI
